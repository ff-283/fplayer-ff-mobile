# 移动/网页播放端设计与实现

## 一、Flutter 架构设计

本项目采用 Flutter + Dart 3.3+ 构建跨平台客户端，Material 3 设计系统，单页面应用，`setState` 管理状态，`SharedPreferences` 持久化偏好。

依赖 (`pubspec.yaml`)：`flutter` 框架、`http`（网络请求）、`shared_preferences`（本地持久化）、`media_kit` / `media_kit_video` / `media_kit_libs_video`（播放器栈）。

应用入口完成 Flutter 引擎绑定和播放内核初始化：

```dart
void main() {
  WidgetsFlutterBinding.ensureInitialized();
  MediaKit.ensureInitialized();
  runApp(const MyApp());
}
```

`MyApp` 是根 `StatefulWidget`，管理三态主题（系统/浅色/深色），内置完整的 Cohere 风格 Material 3 配色（主色 `#17171C`、深绿容器 `#003C33`、辅色蓝 `#1863DC`、石板灰 `#75758A`、错误红 `#B30000`），卡片 22px 圆角无阴影，按钮 32px 药丸形，输入框 8px 圆角聚焦变紫(`#9B60AA`)。

```dart
class MyAppState extends State<MyApp> {
  ThemeMode _themeMode = ThemeMode.system;

  Future<void> _loadThemeMode() async {
    final prefs = await SharedPreferences.getInstance();
    final mode = prefs.getString(_kThemeMode);
    if (!mounted) return;
    setState(() {
      _themeMode = mode == 'light' ? ThemeMode.light
                 : mode == 'dark' ? ThemeMode.dark
                 : ThemeMode.system;
    });
  }

  Future<void> setThemeMode() async {
    final prefs = await SharedPreferences.getInstance();
    setState(() {
      _themeMode = _themeMode == ThemeMode.system ? ThemeMode.light
                 : _themeMode == ThemeMode.light ? ThemeMode.dark
                 : ThemeMode.system;
    });
    await prefs.setString(_kThemeMode, _themeMode.name);
  }
}
```

核心页面 `HttpFlvPlayerPage` 集中管理播放器、URL 解析、日志和偏好设置。页面使用 `Scaffold` + `SafeArea` + `ListView` 实现可滚动布局，视频区 `maxWidth: 800`，表单区 `maxWidth: 600`，日志面板固定 240px 高度，`ListView.builder` 虚拟化渲染，上限 200 条自动裁剪。偏好设置通过文本控制器 `addListener` 触发自动保存，`initState` 时回填。

```dart
class _HttpFlvPlayerPageState extends State<HttpFlvPlayerPage> {
  late final Player _player;
  late final VideoController _videoController;
  final List<LogEntry> _logs = <LogEntry>[];
  final TextEditingController _urlController = TextEditingController(text: '');
  final TextEditingController _gatewayController = TextEditingController(text: 'http://192.168.1.100:9000');
  final TextEditingController _appController = TextEditingController(text: 'live');
  final TextEditingController _streamController = TextEditingController(text: 'stream001');

  String? _error;
  bool _loading = false;
  bool _isStreaming = false;
  UrlInputMode _inputMode = UrlInputMode.service;
  ServicePlayProtocol? _servicePlayProtocol;
}
```

## 二、播放器集成

播放器基于 `media_kit` 系列（底层 libmpv），在 Android 上通过 CMake 编译 arm64-v8a / x86_64 原生库链接进 APK，Windows 通过 CMakeLists.txt 链接 libmpv，Web 降级使用浏览器原生媒体能力。

`initState` 中创建 `Player` 和 `VideoController`，`dispose` 中释放全部资源：

```dart
@override
void initState() {
  super.initState();
  _servicePlayProtocol = (kIsWeb || _isAndroidRuntime) ? ServicePlayProtocol.hls : ServicePlayProtocol.httpFlv;
  _player = Player();
  _videoController = VideoController(_player);
  _bindPlayerLogs();
  _loadSettings();
}

@override
void dispose() {
  _urlController.dispose(); _gatewayController.dispose();
  _appController.dispose(); _streamController.dispose();
  _logScrollController.dispose(); _player.dispose();
  super.dispose();
}
```

自适应控件通过 `MaterialDesktopVideoControlsTheme`（桌面端：播放/暂停 + 音量 + 全屏）与 `MaterialVideoControlsTheme`（移动端：仅全屏）嵌套 + `AdaptiveVideoControls` 分发实现，均关闭 SeekBar（面向实时流而非点播）。

播放状态通过流式监听写入日志：

```dart
void _bindPlayerLogs() {
  _player.stream.playing.listen((p) => _addLog('状态 playing=$p'));
  _player.stream.buffering.listen((b) => _addLog('状态 buffering=$b'));
  _player.stream.completed.listen((c) { if (c) _addLog('流播放完成'); });
}
```

拉流核心：服务模式下先解析地址再播放，直接模式下直接播放。播放前做 Web/Android 协议兼容性拦截：

```dart
Future<void> _play() async {
  if (_isStreaming || _loading) return;
  String url = _urlController.text.trim();
  if (_inputMode == UrlInputMode.service) {
    try { url = await _resolvePlayUrlFromGateway(); _urlController.text = url; }
    catch (e) { setState(() => _error = '解析播放地址失败: $e'); return; }
  }
  if (url.isEmpty) { setState(() => _error = '请输入播放地址'); return; }
  if (kIsWeb && _looksLikeFlvOrRtmp(url)) {
    setState(() => _error = 'Web 环境不支持 FLV/RTMP，请切换 HLS'); return;
  }
  setState(() { _error = null; _loading = true; _isStreaming = true; });
  try {
    await _player.open(Media(url), play: true);
  } catch (e) {
    setState(() { _error = '播放失败: $e'; _isStreaming = false; });
  } finally { if (mounted) setState(() => _loading = false); }
}
```

停止时分两步：`_player.stop()` 终止流 + `_player.pause()` 暂停播放器，最后重置状态标志。

## 三、双模式播放

应用通过 `SegmentedButton` 提供两种入口：「服务地址模式」（通过 Gateway 解析）和「直接 URL 模式」（手输地址播放）。

模式定义：

```dart
enum UrlInputMode { service, direct }
enum ServicePlayProtocol { hls, httpFlv, rtmp }
```

服务模式下的配置卡片包含网关地址、App/Stream 并排输入框和协议分段选择器（HLS / FLV / RTMP）。URL 输入框在服务模式下设为只读（`readOnly: true`），内容由解析结果回填；直接模式下可自由编辑。协议选择具有平台感知——Web 和 Android 默认 HLS，Windows 默认 HTTP-FLV。

FLV/RTMP 检测：

```dart
bool _looksLikeFlvOrRtmp(String url) {
  final v = url.toLowerCase();
  return v.startsWith('rtmp://') || v.contains('.flv');
}
```

Web 环境直接拦截拒绝；Android 仅告警不阻止。

## 四、网络交互

使用 `http` 包，仅涉及与 `fplayer-ff-service` Gateway 的一条接口：

```
GET {gateway}/api/v1/streams/resolve?app={app}&stream={stream}
Accept: application/json
```

核心解析方法：

```dart
Future<String> _resolvePlayUrlFromGateway() async {
  final gateway = _gatewayController.text.trim().replaceAll(RegExp(r'/+$'), '');
  final app = _appController.text.trim();
  final stream = _streamController.text.trim();
  if (gateway.isEmpty || app.isEmpty || stream.isEmpty) throw Exception('请填写服务地址、App、Stream');

  final uri = Uri.parse('$gateway/api/v1/streams/resolve')
      .replace(queryParameters: {'app': app, 'stream': stream});

  final resp = await http.get(uri, headers: {'Accept': 'application/json'});
  if (resp.statusCode < 200 || resp.statusCode >= 300)
    throw Exception('gateway 响应 ${resp.statusCode}: ${resp.body}');

  final decoded = jsonDecode(resp.body);
  if (decoded is! Map<String, dynamic>) throw Exception('响应格式无效');

  final hls = (decoded['playHls'] ?? '').toString().trim();
  final flv = (decoded['playHttpFlv'] ?? '').toString().trim();
  final playUrls = (decoded['playUrls'] is Map<String, dynamic>) ? decoded['playUrls'] : <String, dynamic>{};
  final rtmp = (playUrls['rtmp'] ?? '').toString().trim();
  final hlsFromPlayUrls = (playUrls['hls'] ?? '').toString().trim();

  final selected = _servicePlayProtocol == ServicePlayProtocol.rtmp ? rtmp
    : _servicePlayProtocol == ServicePlayProtocol.hls ? (hls.isNotEmpty ? hls : hlsFromPlayUrls)
    : flv;
  if (selected.isEmpty) throw Exception('响应中缺少对应协议地址');
  return selected;
}
```

JSON 提取逻辑：HLS 优先 `playHls` → 回退 `playUrls.hls`；HTTP-FLV 取 `playHttpFlv`；RTMP 取 `playUrls.rtmp`。异常覆盖参数缺失、HTTP 非 2xx、JSON 格式无效、协议地址缺失四种情况。

Android 网络配置（`AndroidManifest.xml` 关键项）：

```xml
<uses-permission android:name="android.permission.INTERNET" />
<application android:usesCleartextTraffic="true">
```

`usesCleartextTraffic="true"` 是必需配置，否则 Android 阻止所有 HTTP 明文请求导致无法访问局域网的 `http://` 地址。`MainActivity` 直接继承 `FlutterActivity()`，使用 V2 embedding。

## 五、多平台打包

支持 Android、Windows、Web 三平台，通过 `scripts/build_release.ps1` 统一打包。

Android 构建基于 Gradle Kotlin DSL，关键配置：Java 17 编译、`arm64-v8a` + `x86_64` ABI、条件签名策略（有 `key.properties` 则正式签名，否则 debug 签名）、产物 APK + AAB。应用图标由 `flutter_launcher_icons` 从 `doc/img/icon.png` 生成。

`android/app/build.gradle.kts` 签名部分：

```kotlin
val keystorePropertiesFile = rootProject.file("key.properties")
val keystoreProperties = Properties()
if (keystorePropertiesFile.exists()) { keystorePropertiesFile.inputStream().use { keystoreProperties.load(it) } }

ndk { abiFilters.addAll(listOf("arm64-v8a", "x86_64")) }

buildTypes {
    release {
        signingConfig = if (keystorePropertiesFile.exists()) {
            signingConfigs.create("release") {
                storeFile = file(keystoreProperties["storeFile"] as String)
                storePassword = keystoreProperties["storePassword"] as String
                keyAlias = keystoreProperties["keyAlias"] as String
                keyPassword = keystoreProperties["keyPassword"] as String
            }
        } else { signingConfigs.getByName("debug") }
    }
}
```

Windows 构建基于 CMake + MSBuild，`CMakeLists.txt` 编译 `main.cpp`、`flutter_window.cpp`、`win32_window.cpp` 等源文件，链接 `flutter`、`flutter_wrapper_app`、`dwmapi.lib`，通过 `add_dependencies(flutter_assemble)` 确保 Dart 编译在前。`main.cpp` 入口初始化 COM 环境后创建 1280x720 窗口，进入标准 Win32 消息循环。

打包脚本 `build_release.ps1` 核心流程：

```powershell
param([string]$BuildName, [string]$BuildVersionName, [string]$BuildVersionNumber,
      [switch]$SkipWindows, [switch]$SkipAndroid, [switch]$SkipApk, [switch]$SkipAab,
      [switch]$RunAnalyze, [switch]$RunTest)

flutter pub get
$buildArgs = @("--release")
if ($BuildVersionName) { $buildArgs += "--build-name"; $buildArgs += $BuildVersionName }
if ($BuildVersionNumber) { $buildArgs += "--build-number"; $buildArgs += $BuildVersionNumber }

if (-not $SkipWindows) { flutter build windows @buildArgs }
if (-not $SkipAndroid) {
    if (-not $SkipApk) { flutter build apk @buildArgs }
    if (-not $SkipAab) { flutter build appbundle @buildArgs }
}

# 产物收集到 dist/<时间戳>/，生成 checksums.txt + release-notes-template.md
```

产物目录结构：Windows 产物打包为 zip，Android APK/AAB 直接复制，附带 SHA256 校验文件和发布说明模板。脚本支持 `-RunAnalyze` / `-RunTest` 在构建前执行静态分析和单元测试。
