# UI 视觉与交互优化说明

## 概述

对 `lib/main.dart` 进行了全面的视觉和交互体验升级，核心方向：**深色专业风格 + 细腻动效 + 清晰信息层级**。

---

## 设计主题

```
背景色：#0D1117 (Scaffold) → #010409 (底部渐变)
表面色：#161B22 (Card / 日志标题栏)
边框色：#30363D
主色调：#58A6FF (蓝)
强调色：#3FB950 (绿)
危险色：#F85149 (红)
警告色：#D29922 (黄)
```

全局通过 `ThemeData(brightness: Brightness.dark)` + 自定义 `ColorScheme` 统一生效，所有组件（TextField、Button、Card、SnackBar、SegmentedButton）均在主题层统一样式，无需逐个覆盖。

---

## 各区块改动详情

### 1. AppBar

- **渐变图标**：左侧 32×32 圆角方块，蓝绿渐变底 + 白色播放图标，作为应用标识
- **标题**：中文"局域网流播放器"，字重 600
- **深色渐变背景**：`#161B22 → #0D1117`
- **LIVE 状态徽标**：流播放时右侧显示绿色圆点 + "LIVE" 文字徽标，`_StreamingBadge` 独立组件

### 2. 视频播放区

- **容器**：12px 圆角 + 边框，闲置时灰色边框，播放中边框高亮蓝色 `#388BFD` 并降低透明度
- **呼吸光晕**：播放中通过 `AnimationController`（1500ms 循环）驱动 `BoxShadow`，产生蓝色外发光脉冲效果
- **占位态**：未播放时显示半透明电视图标 + "等待播放" 文字
- **加载态**：播放中有半透明 spinner 覆盖

### 3. 模式切换

- SegmentedButton 添加图标（服务器 / 链接）提升辨识度
- 标签精简为"服务模式" / "直接链接"，避免文字溢出
- 去掉 `maxWidth: 600` 约束，按钮撑满可用宽度

### 4. 服务配置卡片

- 整个服务配置区封装为 `Card` 组件，带标题行（云端图标 + "服务配置"）
- 三个输入框增加 prefixIcon（language / apps / stream）
- 协议选择与标题同行的右侧区域，紧凑布局
- 卡片使用 `AnimatedSize` 实现切换模式时的平滑展开/收起

### 5. URL 输入

- prefixIcon 根据模式切换：服务模式显示自动解析图标，直接模式显示编辑图标
- 右侧清除按钮（URL 非空时显示）
- 只读状态下文字自动变灰（`colors.tertiary`）

### 6. 操作按钮

- **拉流按钮**：绿色 `FilledButton.icon`，带播放图标，加载中切换为 spinner + "连接中..."
- **停止按钮**：红色 `OutlinedButton.icon`，带停止图标
- `AnimatedSwitcher` 平滑切换按钮文字/图标状态
- `IntrinsicHeight` + `minimumSize: Size.fromHeight(48)` 确保两按钮严格等高等宽
- 水平 padding 统一为 20px

### 7. 错误提示

- 卡片式展示，半透明红色背景 + 红色边框
- 左侧错误图标，右侧关闭按钮可手动消除

### 8. 日志面板

完整的终端模拟器风格：

| 区域 | 说明 |
|------|------|
| **标题栏** | 深色背景 `#161B22`，终端图标 + "日志" 标题 + 三色计数徽标 |
| **计数徽标** | INFO(蓝) / WARN(黄) / ERROR(红)，实时显示各级日志数量 |
| **操作按钮** | 复制 / 清空，无日志时自动置灰 |
| **日志行** | 等宽字体，时间戳(灰) + 级别标签(彩色圆角方块) + 消息内容(白/灰) |
| **空状态** | 居中"暂无日志"提示 |

### 9. 动画系统

| 动画 | 实现方式 | 触发条件 |
|------|----------|----------|
| 视频容器呼吸光晕 | `AnimationController` + `Tween<double>` 循环 | 播放中 |
| 按钮状态切换 | `AnimatedSwitcher` | 拉流/加载中状态切换 |
| 服务配置展开 | `AnimatedSize` | 模式切换时 |
| 日志自动滚动 | `ScrollController.animateTo` + `Curves.easeOut` | 新日志写入 |

## 交互优化要点

- **按钮状态互斥**：拉流时停止按钮可用、拉流按钮禁用（反之亦然），避免误操作
- **URL 自动解析**：服务模式下 URL 输入框只读，配合自动解析图标提示用户来源
- **即时反馈**：SnackBar 浮动样式（`SnackBarBehavior.floating`），日志复制操作有即时提示
- **剪贴板支持**：SelectionArea 包裹日志列表，支持直接选中复制
- **调试信息可见**：右上角 LIVE 徽标提供实时播放状态

`lib/main.dart` — UI 层全部改动，业务逻辑（播放/停止/解析/日志绑定）保持不变。
